01 — 문제 진단
왜 CS 문의가 반복되는가
이건 "설명 부족" 문제가 아니라 구조 문제입니다. 안내문을 더 길게 쓰거나 빨간 글씨를 추가하는 것으로는 해결하기 어렵습니다.
구조적 문제 1

PPT가 설명 도구를 넘어서 프로세스 허브 역할까지 하고 있습니다

강의자료 비밀번호, 리워드 암호, 신청서 링크, 다운로드 링크가 VOD 설명 → 영상 내부 → 압축파일 → PPT → 구글 폼 → 이메일로 분산되어 있어 수강생이 전체 흐름을 한 번에 이해하기 어렵습니다.

구조적 문제 2

밤샘챌린지의 복잡한 흐름이 강의자료 비밀번호와 섞여 혼동을 배가시킵니다

수강생 입장에서는 둘 다 "뭐 열 때 넣는 숫자"일 뿐인데, 하나는 영상에서, 다른 하나는 신청서 제출 후 메일에서 옵니다. PPT 안에 신청서 작성과 리워드 받기 버튼이 나란히 보이니 순서를 어기는 UI가 됩니다.

구조적 문제 3

리워드 암호가 메일 기반이라 스팸함 이탈이 구조적으로 발생합니다

Zapier를 통해 자동 발송되는 리워드 암호 메일이 스팸함으로 빠지는 경우가 빈번합니다. "스팸함을 확인해 보세요"라고 CS 응대를 해도, 자동 발송 메일은 스팸 필터에 걸리기 쉬운 구조입니다.

참고: 강의자료 비밀번호의 목적 — 강의 수강 전 자료 열람을 차단하여 환불을 방어하기 위함입니다. 본 개선안은 이 "수강 전 자료 열람 차단" 기능을 유지하는 방향으로 설계했습니다.

02 — 개선안
미션 허브 + 구글 폼 개편
흩어져 있는 정보를 한 화면에 모으고, 메일 발송을 없애는 것. 미션 허브는 GitHub Pages에 올리는 정적 HTML 한 장입니다. 별도 서버나 개발자가 필요 없고, 바이브코딩으로 구축 가능합니다.
S1

STEP 1: 강의자료 다운로드 존 (미션 허브)

해결하는 문제: #1, #2, #3, #5

구조

  • "VOD 영상에 등장한 패스워드를 입력하세요" 입력창을 둡니다
  • 올바른 암호 입력 시 → 구글 드라이브 다운로드 링크가 노출됩니다
  • 압축파일은 유지 (PPT + 부가 자료 묶음). 단, 압축파일 자체의 비밀번호는 제거하고 미션 허브의 웹 암호가 "수강 전 자료 열람 차단" 역할을 대신합니다

오프라인/온라인 반 통합 운영

  • 오프라인 반: 토요일 현장에서 받은 암호로 미션 허브에서 바로 다운로드
  • 온라인 반: VOD 속 암호를 미션 허브에 입력하여 다운로드
  • 동일한 허브 구조로 오프라인/온라인 통합 운영 가능

운영 효율 개선

  • 현재: 기수/주차마다 압축파일 생성 → 드라이브 업로드 → VOD 설명에 링크 첨부 (반복 작업)
  • 개선: 구글 드라이브에 파일을 한 번만 올려두고, 미션 허브의 암호/링크만 변경하면 됨

핵심 변경: 구글 폼 섹션 분기로 메일 발송 제거

  • 구글 폼의 "응답에 따라 섹션으로 이동" 기능을 활용
  • 성공 경로의 마지막 섹션에서 리워드 다운로드 링크를 직접 제공
  • 리워드 파일은 압축 없이 직접 다운로드 링크로 제공 → 리워드 암호 자체가 불필요
  • 구글 폼 설정 변경만으로 구현 가능 (개발 불필요)
