01 — 문제 진단
밤샘챌린지 리워드 수령 과정의 구조적 문제
밤샘챌린지 리워드 수령 과정에서 발생하는 반복 CS는 안내 부족이 아닌 구조적 문제입니다. 메일 스팸함, 비밀번호 혼동, 이메일 오기재 등으로 수강생의 참여 의지를 꺾고, 운영팀의 에너지를 소모시킵니다.
구조적 문제 1

강의자료 비밀번호와 리워드 암호가 혼동됩니다

수강생 입장에서 둘 다 '무언가를 열 때 넣는 숫자'입니다. 하나는 강의 영상에서, 다른 하나는 메일에서 오기 때문에 혼동이 구조적으로 발생합니다. "이 비번 넣었는데 안 돼요"라는 CS가 반복됩니다.

구조적 문제 2

리워드 암호 메일이 스팸함으로 빠집니다

Zapier를 통해 자동 발송되는 리워드 암호 메일이 스팸함으로 빠지는 경우가 빈번합니다. "스팸함을 확인해 보세요"라고 안내해도, 자동 발송 메일은 스팸 필터에 걸리기 쉬운 구조입니다.

구조적 문제 3

구글 폼으로는 성공/실패 분기가 불가능합니다

구글 폼은 제출 완료 후 최종 화면을 성공/실패별로 다르게 설정할 수 없습니다. 성공자(실패 후 업무일지 10개 작성 포함)에게만 리워드를 제공하는 분기 처리가 구조적으로 불가능하며, 업무일지 10개 정책도 검증할 수 없어 사문화되어 있습니다.

02 — 개선안
밤샘챌린지 전용 페이지 구축
PPT 안의 기존 구글폼 링크를 전용 페이지 링크로 교체합니다. 수강생은 기존처럼 PPT에서 링크를 클릭하고, 제출 즉시 리워드를 다운로드할 수 있습니다. 별도 서버 비용·개발 인력 없이 운영 가능합니다.
⚡

전용 페이지: 제출 즉시 리워드 다운로드

구글폼 + Zapier 메일 → 자체 폼 + Google Apps Script로 대체

수강생 동선 비교

  • 변경 전: PPT에서 구글폼 링크 → 구글폼 작성 → 메일 대기 → 메일에서 암호 확인 → PPT에서 리워드 열기 → 암호 입력
  • 변경 후: PPT에서 전용 페이지 링크 → 신청서 작성 + 제출 → 리워드 바로 다운로드

기술 구조

  • 전용 페이지에 이름, 연락처, 성공/실패 선택, 업무일지 링크 입력 폼을 직접 구축
  • [제출] 버튼 클릭 시 제출 데이터가 구글 스프레드시트에 자동 기록
  • 스프레드시트 기록이 완료된 뒤에만 리워드 다운로드 링크가 화면에 표시
  • 제출 전에는 리워드 링크가 화면 어디에도 보이지 않으므로, 미제출 상태에서의 링크 탈취가 불가능
  • 별도 서버 없이 운영 가능하며, 제출 데이터는 구글 스프레드시트에서 바로 확인 가능

리워드 수령 분기

  • [성공] 업무일지 1개 제출 → [제출] 클릭 → 리워드 즉시 다운로드
  • [실패 + 업무일지 10개] 10개 링크를 개별 입력란에 모두 입력 → [제출] 클릭 → 리워드 즉시 다운로드
  • [실패] [제출] 클릭 → 격려 메시지 표시

PPT 변경 사항

  • 기존: '밤샘챌린지 신청서 작성' 버튼 (구글폼 링크) + '리워드 받기' 버튼
  • 변경: '밤샘챌린지 제출 & 리워드 받기' 버튼 1개 (전용 페이지 링크) + '제출 즉시 리워드를 바로 받을 수 있습니다 (메일 대기 없음)' 안내 문구

어뷰징 방지

  • [제출] 버튼 클릭 시 '업무일지 허위 작성 등 어뷰징이 확인될 경우 해당 기수의 모든 리워드 및 혜택에서 제외될 수 있습니다' 확인 팝업 표시

03 — 변경 범위
변경하는 것과 변경하지 않는 것
변경 범위를 밤샘챌린지 리워드 수령 과정에 한정하여, 기존 운영 방식에 미치는 영향을 최소화합니다.

🔄 변경하는 것

  • → 밤샘챌린지 제출 경로 (구글폼 → 전용 페이지)
  • → 리워드 수령 방식 (메일 암호 → 제출 즉시 다운로드)
  • → PPT 안의 링크 (구글폼 → 전용 페이지)
  • → Zapier 메일 발송 비활성화
  • → 업무일지 10개 정책 시스템화

✅ 변경하지 않는 것

  • → 강의자료 다운로드 방식 (현행 유지)
  • → 강의자료 비밀번호 구조 (현행 유지)
  • → VOD / 현장 강의 운영 방식
  • → 수강 유도 장치 (암호 게이트)
  • → 카카오톡 단톡방 운영 방식
핵심: 수강생의 기존 동선에서 달라지는 건 'PPT에서 링크를 누르면 구글폼 대신 전용 페이지가 열린다'는 것과 '메일을 기다리지 않아도 된다'는 것뿐입니다.
04 — 기대효과
무엇이 달라지는가
수강생·운영팀 두 측면에서의 기대효과입니다.
🎓

수강생: 리워드 수령이 간단해짐

메일 대기·스팸함 확인 과정이 사라져 제출하면 바로 리워드 수령. 비밀번호 2개→1개 축소로 혼동 원천 차단. 챌린지 제출 허들이 낮아져 참여율 향상 기대. 실패 수강생에게 업무일지 10개 경로를 통한 추가 학습 동기 부여.

⚙️

운영팀: 반복 CS 원천 제거

'리워드 메일이 안 왔어요' → 메일 자체가 없으므로 원천 차단. '리워드 비번이 뭔가요?' → 리워드 암호가 없으므로 원천 차단. '강의자료 비번이랑 헷갈려요' → 리워드 비번 자체가 없으므로 원천 차단. 수강생별 참여 현황이 스프레드시트에 자동 기록되어 선제적 케어 가능.
