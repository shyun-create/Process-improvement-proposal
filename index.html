01 — 문제 진단
수강 성과를 저해하는 구조적 요인
비밀번호 혼동·메일 누락 등으로 발생하는 반복 CS는 단순한 설명 부족이 아닌 구조적 문제입니다. 수강생의 학습 흐름을 끊고, 운영팀의 에너지를 소모시키며, 양쪽 모두 '강의 내용'이 아닌 '시스템 문제'에 시간을 쓰게 만듭니다.
구조적 문제 1

PPT가 설명 도구를 넘어서 프로세스 허브 역할까지 하고 있습니다

강의자료 비밀번호, 리워드 암호, 신청서 링크, 다운로드 링크가 VOD 설명 → 영상 내부 → 압축파일 → PPT → 구글 폼 → 이메일로 분산되어 있어 수강생이 전체 흐름을 한 번에 이해하기 어렵습니다.

구조적 문제 2

밤샘챌린지의 복잡한 흐름이 강의자료 비밀번호와 섞여 혼동을 배가시킵니다

수강생 입장에서는 둘 다 "뭐 열 때 넣는 숫자"일 뿐인데, 하나는 영상에서, 다른 하나는 신청서 제출 후 메일에서 옵니다. PPT 안에 신청서 작성과 리워드 받기 버튼이 나란히 보이니 순서를 어기는 UI가 됩니다.

구조적 문제 3

리워드 암호가 메일 기반이라 스팸함 이탈이 구조적으로 발생합니다

Zapier를 통해 자동 발송되는 리워드 암호 메일이 스팸함으로 빠지는 경우가 빈번합니다. "스팸함을 확인해 보세요"라고 CS 응대를 해도, 자동 발송 메일은 스팸 필터에 걸리기 쉬운 구조입니다.

참고: 강의자료 비밀번호의 목적 — 수강 전 자료 차단 (환불 방어 목적)입니다. 본 개선안은 이 "수강 전 자료 열람 차단" 기능을 현행 암호 게이트 그대로 유지하면서, 미션 허브 내 4개 접점(공지사항·암호 입력 확인 알림·FAQ·리워드 수령 경고)에서 '강의를 반드시 먼저 수강해야 한다'는 안내를 반복 배치하여 경각심을 부여하는 방향으로 설계했습니다.

02 — 개선안
미션 허브 구축
현행 수강 유도 장치(암호 게이트)는 그대로 유지하면서, 공지사항·확인 알림·FAQ·리워드 수령 경고 등 다중 안내 장치를 통해 경각심을 지속적으로 부여합니다. 암호 인증 이후의 자료 수령 동선은 간소화하고, 구글 폼의 구조적 한계는 자체 폼을 구축하여 해결하였으며, 기존에 사문화되어 있던 운영 정책(업무일지 10개)을 실제 작동하는 시스템으로 전환합니다. 별도 개발 인력·서버 비용 없이 운영 가능합니다.
S1

STEP 1: 강의자료 다운로드 (미션 허브 · 강의자료 탭)

완화하는 문제: #1 · 해결하는 문제: #2, #3 · 개선하는 문제: #5

구조

  • 공지사항 확인 체크: 미션 허브 상단에 '강의자료 비밀번호는 강의를 수강해야만 확인할 수 있습니다. 반드시 강의를 먼저 수강한 뒤 이용해 주세요.' 공지를 배치하고, '위 공지사항을 확인했습니다' 체크를 완료해야 다운로드 버튼과 리워드 수령 버튼이 활성화됩니다
  • 암호 입력 시 수강 확인 알림: 올바른 암호를 입력하고 확인 버튼을 누르면, 다운로드 링크가 바로 노출되지 않고 '강의를 끝까지 수강하셨나요? 강의자료를 제대로 활용하기 위해서는 반드시 강의를 먼저 수강해야 합니다.' 확인 알림이 먼저 표시됩니다. 수강생이 '확인'을 누른 뒤에만 다운로드 링크가 열립니다
  • 압축파일은 유지 (PPT + 부가 자료 묶음). 단, 압축파일 자체의 비밀번호는 제거하고 미션 허브의 암호 게이트가 수강 전 파일 노출을 줄이는 장벽 역할을 합니다

수강 필수 안내 — 다중 장치를 통한 경각심 부여

  • ① 미션 허브 상단 공지사항: 페이지 진입 시 가장 먼저 보이는 위치에 강의 수강 필수 안내를 배치하고, 확인 체크를 완료해야 다운로드·리워드 수령 버튼이 활성화
  • ② 암호 입력 직후 확인 알림: 올바른 암호를 입력해도 바로 다운로드되지 않고, '강의를 끝까지 수강하셨나요?' 확인 알림을 한 번 더 표시
  • ③ FAQ 안내: '강의자료 암호는 어디서 확인하나요?' 질문에 '반드시 강의를 수강해야만 확인할 수 있습니다' 답변을 명시
  • ④ 리워드 수령 직전 경고: 밤샘챌린지 제출 시 '업무일지 허위 작성 등 어뷰징이 확인될 경우 해당 기수의 모든 리워드 및 혜택에서 제외될 수 있습니다' 확인 팝업을 표시

진입 경로: 카카오톡 수강생 전용 단톡방

  • 매주 강의 전 카카오톡 수강생 전용 단톡방에 미션 허브 링크를 공지합니다
  • 단톡방은 수강생 전원이 참여하고 있고 실시간 확인율이 가장 높은 채널이므로, VOD 상세설명에 링크를 묻어두는 방식보다 도달률이 높습니다
  • VOD 상세설명에도 백업으로 링크를 배치합니다

오프라인/온라인 반 통합 운영

  • 오프라인 반: 현장 강의에서 확인한 암호로 미션 허브에서 바로 다운로드
  • 온라인 반: VOD 속 암호를 미션 허브에 입력하여 다운로드
  • 동일한 허브 구조로 오프라인/온라인 통합 운영 가능

파일 공유 방식

  • 마스터 폴더가 아닌 주차별 개별 파일 링크로 공유하여, 유출 시에도 해당 주차 자료만 노출되도록 합니다
  • 구글 드라이브 공유 설정은 "링크가 있는 모든 사용자 — 뷰어"를 유지하되, 미션 허브의 암호 게이트가 링크 도달 전 장벽 역할을 합니다

운영 효율 개선

  • 현재: 기수/주차마다 ZIP 생성 → 비밀번호 설정 → 드라이브 업로드 → VOD 설명에 링크 첨부 (반복 작업)
  • 개선: 기수 및 주차별로 고유 URL을 생성하여 배포합니다 (예: .../27-1, .../27-2). 이를 통해 이전 주차를 늦게 수강하는 인원의 접근성도 완벽하게 보장하며, ZIP 생성·비밀번호 설정 등 반복 단계가 줄어듭니다. 페이지 복제 및 URL 생성은 주차별 복제·링크 교체 중심으로 운영 부담을 최소화할 수 있습니다

STEP 2: 밤샘챌린지 리워드 (미션 허브 · 자체 폼)

해결하는 문제: #3, #4

왜 구글 폼을 쓰지 않는가

  • 구글 폼은 제출 완료 후 확인 메시지(엔딩 페이지)를 성공/실패 경로별로 다르게 설정할 수 없는 구조적 한계가 있습니다
  • 성공 경로에 리워드 링크를 넣어도, 실패자에게 동일한 최종 화면이 노출되어 성공자에게만 리워드를 제공하는 분기 처리가 불가능합니다
  • 이 한계를 근본적으로 해결하기 위해, 구글 폼 대신 미션 허브 내에 자체 신청 폼을 구축했습니다

핵심 변경: 미션 허브 자체 폼 + 스프레드시트 자동 기록

  • 미션 허브의 밤샘챌린지 탭에 이름, 연락처, 성공/실패 선택, 업무일지 링크 입력 폼을 직접 구축
  • 수강생이 [제출] 버튼을 클릭하면 제출 데이터가 구글 스프레드시트에 자동 기록됩니다
  • 스프레드시트 기록이 완료된 뒤에만 리워드 다운로드 링크가 화면에 표시됩니다
  • 제출 전에는 리워드 링크가 화면 어디에도 보이지 않으므로, 성공자에게만 리워드를 제공하는 분기 처리가 완벽하게 구현됩니다
  • 별도 서버 비용 없이 운영 가능하며, 제출 데이터는 구글 스프레드시트에서 바로 확인 가능

리워드 수령 분기 구조

1
공통: 이름, 연락처 입력 + 성공/실패 선택

"성공했어요" 선택 시 → 업무일지 링크 1개 입력란 표시

"실패했어요" 선택 시 → 업무일지 10개 작성 여부 질문 표시

2
성공 경로: 업무일지 1개 제출 → 리워드 즉시 다운로드

[제출] 클릭 → 스프레드시트 기록 → 리워드 다운로드 링크 표시

3
실패 + 업무일지 10개 경로: 10개 링크 제출 → 리워드 즉시 다운로드

밤샘챌린지에 실패하더라도, 업무일지를 10개 작성하여 제출하면 리워드 수령 가능

4
실패 경로: 격려 메시지

업무일지 10개를 작성하지 않은 경우, 격려 메시지와 함께 제출 완료

이렇게 하면

  • Zapier 메일 발송 + 구글 폼 모두 제거 → 스팸함 문제 원천 차단
  • 리워드 암호가 사라짐 → 강의자료 비밀번호와의 혼동 원천 차단
  • 제출 완료 전까지 리워드 링크 비노출 → 성공/실패별 서로 다른 최종 결과 화면 제공 가능
  • 실패해도 업무일지 10개 제출 시 리워드 수령 가능 → 수강생 동기부여 유지
  • 제출 데이터가 구글 스프레드시트에 자동 기록 → 운영팀 모니터링 용이
  • 어뷰징이 확인될 경우 해당 기수의 모든 리워드 및 혜택에서 제외될 수 있다는 경고 문구를 제출 시 확인 팝업으로 표시하여 부정 수령을 억제

04 — 수강 성과 영향 검토
수강 유도: 현행 장치 유지 + 다중 안내 장치로 경각심 부여
본 제안이 수강 유도와 학습 성과에 미치는 영향을 항목별로 검토하였습니다.
✅

우려 해소: '수강 없이 자료부터 열람하는 행위'

→ 현행 암호 게이트를 유지하고, 4개 접점에서 수강 필수 안내를 반복 배치하여 경각심 부여

강의를 수강해야만 암호를 알 수 있는 현행 구조를 그대로 유지합니다. 여기에 미션 허브 상단 공지사항, 암호 입력 직후 확인 알림, FAQ, 리워드 수령 직전 경고 — 4개 접점에서 '강의를 반드시 먼저 수강해야 한다'는 안내를 반복 배치하여, 수강생이 미션 허브를 이용하는 전 과정에서 경각심을 유지하도록 설계했습니다. 단일 안내가 아닌 다중 접점에서의 반복 안내이므로, 무의식적으로 넘기기 어려운 구조입니다.

✅

우려 해소: '자료를 먼저 열람하면 공지사항 전달이 안 됨'

→ 공지 확인 체크 후에만 다운로드 진행 가능

미션 허브 상단에 주차별 공지 배너를 배치하고, "공지사항을 확인했습니다" 체크박스를 통과해야 다음 단계로 진행 가능합니다. 현행 대비 공지 도달률이 구조적으로 향상됩니다.

✅

긍정: 기존 정책의 실현을 통한 수강 성과 기여

→ 사문화된 정책이 실제 작동하여 추가 학습 동기 부여

업무일지 10개 제출 정책이 기술적으로 작동하게 되어, 밤샘챌린지에 실패한 수강생에게도 구체적인 학습 동기를 부여합니다. 제출 과정 간소화로 참여율 향상이 기대되며, 참여율은 수강 성과와 직결되는 지표입니다.

✅

긍정: 강의자료와 밤샘챌린지의 동일 페이지 배치로 챌린지 인지율 향상

→ 별도 안내 없이도 챌린지 참여로 이어지는 구조

기존에는 밤샘챌린지 신청 경로가 PPT 내 구글 폼 링크에 한정되어, 챌린지 존재를 인지하지 못하거나 나중에 하려다 잊어버리는 케이스가 있었습니다. 미션 허브에서는 강의자료를 받으러 온 수강생이 바로 옆 탭에서 밤샘챌린지를 확인할 수 있어, 자연스러운 교차 노출이 발생합니다. 이는 챌린지 참여율 향상으로 이어질 수 있으며, 참여율은 수강 성과와 직결되는 지표입니다.

05 — 기대효과
무엇이 달라지는가
수강생·운영팀·수강 성과 세 측면에서의 기대효과입니다.
🎓

수강생: 학습에 집중할 수 있는 환경

비밀번호 2개→1개 축소, 메일 대기·스팸함 확인 과정이 사라져 수강생이 강의 내용에만 집중 가능. 챌린지 제출 허들이 낮아져 참여율 향상 기대. 강의자료와 밤샘챌린지가 동일 페이지에 위치하여, 별도 안내 없이도 챌린지 인지율이 구조적으로 향상. 실패 수강생에게 업무일지 10개 경로를 통한 추가 학습 동기 부여.

⚙️

운영팀: 반복 CS에서 성과 관리로 전환

비밀번호·메일 관련 반복 CS 제거로 응대 에너지를 수강생 관리에 집중 가능. 수강생별 챌린지 참여 현황이 스프레드시트에 자동 기록되어 미참여 수강생 식별 및 선제적 케어 가능.

📈

수강률 및 만족도 향상

불필요한 시스템 마찰이 사라지면 수강생은 '시스템이 복잡하다'가 아닌 '강의 내용이 좋다'에 집중. 참여 데이터 기반의 선제적 케어로 중도 이탈 감소. 수강 저해 요인 감소로 만족도 개선 가능성 확보.

07 — 실행 로드맵
어떻게 진행할 것인가
1
승인 후 즉시 실행

미션 허브 구축 + 기존 PPT 정리

미션 허브 페이지: 별도 개발 인력 없이 자체 구현 완료 (프로토타입 구축·테스트 완료). 스프레드시트 연동 완료. Zapier 메일 발송 비활성화. 강의자료 압축파일에서 비밀번호 제거 + 구글 드라이브에 주차별 개별 파일 업로드. 기존 강의자료 PPT 내의 '밤샘챌린지 리워드 신청서 작성' / '리워드 받기' 버튼을 제거하고, '미션 허브에서 신청하세요'라는 안내 문구로 대체 검토.

2
중기 실행 — 27기 파일럿

파일럿 적용 + 모니터링

신규 기수(27기)부터 변경된 프로세스 적용. 비밀번호/리워드 관련 CS 문의를 현행 대비 유의미하게 감소시키는 것이 목표이며, 필요하다면 구체적 수치 목표는 26기 CS 데이터 집계 후 설정 예정. 수강생 피드백 수집 후 프로세스 미세 조정.

3
장기 고도화

보안 검증 고도화 (필요 시)

현재 정적 페이지 기반의 최소 장벽을 서버리스 검증 구조로 고도화하여, 소스코드에서 리워드 URL이 완전히 제거되는 구조로 전환. 별도 상시 서버 비용 없이 운영 가능. 파일럿 운영 결과에 따라 고도화 범위와 시기를 결정.
